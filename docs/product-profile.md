# سلّمها - Sallemha
## منصة تسليم المنتجات الرقمية لمتاجر سلة

---

## الفكرة

**سلّمها** منصة SaaS احترافية تحل مشكلة تسليم المنتجات الرقمية (أكواد، بطاقات، مفاتيح تفعيل، ملفات، روابط، رخص، اشتراكات) لعملاء المتاجر الإلكترونية.

**المشكلة:** التاجر يضطر يسلّم المنتج الرقمي يدويًا عبر واتساب أو رسائل، مما يسبب:
- تأخير في التسليم
- أخطاء في إرسال الأكواد
- عدم وجود إثبات استلام
- تشتت المخزون الرقمي
- ضغط على التاجر والموظفين
- تجربة عميل غير احترافية

**الحل:** التاجر يرفع مخزونه الرقمي في المنصة، وعند كل طلب يتولّد رابط تسليم آمن. العميل يفتح الرابط، يتحقق من هويته، وتظهر له بيانات المنتج الرقمي تلقائيًا مع توثيق كامل للعملية.

---

## رحلة التاجر (Merchant Journey)

### 1. التسجيل
- التاجر يدخل المنصة ويضغط "إنشاء حساب جديد"
- يدخل: الاسم، البريد الإلكتروني، كلمة المرور، اسم المتجر
- يتولّد slug فريد للمتجر (يُستخدم في روابط التسليم)
- يدخل مباشرة للوحة التحكم

### 2. لوحة التحكم (Dashboard)
يشاهد التاجر:
- **بطاقات إحصائية:** إجمالي الطلبات | الطلبات المسلمة | طلبات معلقة | المخزون المتاح
- **آخر الطلبات:** جدول بآخر 10 طلبات مع حالة التسليم لكل طلب
- **تنبيهات المخزون:** المنتجات اللي مخزونها أقل من 5 عناصر (باللون الأحمر)

### 3. إدارة المنتجات الرقمية
- يضغط "منتجات" من القائمة الجانبية
- يضغط "+ منتج جديد"
- يملأ: اسم المنتج، النوع (كود/مفتاح/رابط/ملف/بيانات دخول)، تعليمات الاستخدام للعميل
- يشاهد كل منتج مع:
  - عدد المخزون الكلي
  - عدد المخزون المتاح (بالأخضر)
  - حالة المنتج (نشط/معطل)

### 4. إدارة المخزون
- يفتح منتج معين
- يضيف الأكواد إما:
  - **يدويًا:** كود واحد في كل مرة
  - **نصيًا:** لصق مجموعة أكواد (كل سطر = كود)
  - **ملف CSV:** رفع ملف
- الأكواد تُشفر تلقائيًا (AES-256) ولا تظهر في أي مكان إلا للعميل المستحق
- يقدر يعدّل كود أو يعطّله

### 5. إنشاء الطلبات
- يضغط "الطلبات" من القائمة
- يضغط "+ طلب جديد"
- يملأ: رقم الطلب، اسم العميل، رقم جوال العميل، المنتج المطلوب
- يتولّد تلقائيًا:
  - رابط تسليم آمن (64 حرف عشوائي)
  - raw_token (يظهر مرة واحدة فقط للتاجر)
- التاجر ينسخ الرابط ويرسله للعميل (يدويًا في MVP)
- يشاهد التاجر جدول الطلبات مع:
  - رقم الطلب | اسم العميل | المنتج | حالة التسليم | حالة الرابط
- حالات التسليم:
  - **تم الإرسال** — الرابط مُرسل للعميل
  - **تم الفتح** — العميل فتح الرابط
  - **تم التحقق** — العميل أدخل بيانات صحيحة
  - **تم الاستلام** — العميل شاف المنتج واستلمه

### 6. متابعة التسليم
- التاجر يفتح طلب معين يشوف:
  - بيانات العميل
  - حالة التسليم الحالية
  - حالة الرابط (نشط/معطل)
- يضغط "سجل التوصيل" يشوف:
  - `link_created` — متى تم إنشاء الرابط
  - `link_opened` — متى فتح العميل الرابط
  - `verification_success` — متى تحقق العميل بنجاح
  - `product_viewed` — متى شاف العميل المنتج
  - `link_resent` — متى تم إعادة إرسال الرابط

### 7. إعادة إرسال الرابط
- إذا العميل فقد الرابط أو احتاج رابط جديد
- التاجر يضغط "إعادة إنشاء الرابط"
- يتولّد raw_token جديد ويُعطّل القديم
- **ملاحظة أمنية:** إذا الطلب تم تسليمه (product_viewed)، لا يمكن إعادة توليد الرابط تلقائيًا

### 8. الإعدادات
- تعديل اسم المتجر
- تغيير اللون الرئيسي (ينعكس على صفحة التسليم)
- اختيار طريقة التحقق: رقم الطلب + الجوال، أو رقم الطلب + رمز تحقق

## رحلة العميل (Customer Journey)

### 1. استلام الرابط
- العميل يستلم رابط من التاجر (عبر واتساب، رسالة، إيميل)
- شكل الرابط: `https://sallemha.com/d/{store_slug}/{raw_token}`
- مثال: `https://sallemha.com/d/matjar-ahmad/a1b2c3...`

### 2. فتح الرابط (شاشة 1)
- يفتح الرابط على جواله
- يشوف:
  - شعار المتجر (أول حرف من اسمه إذا ما فيه شعار)
  - اسم المتجر
  - عبارة "صفحة تسليم المنتج الرقمي"
- الحالة: **Loading** ← **نموذج التحقق**

### 3. التحقق من الهوية (شاشة 2)
- يظهر نموذج بسيط:
  - **رقم الطلب** (يدخله العميل)
  - **رقم الجوال** (اللي سجل به الطلب)
- يضغط "تحقق من الطلب"
- الحالات المحتملة:
  - ❌ **بيانات خاطئة** — رسالة حمراء "بيانات التحقق غير صحيحة"
  - ❌ **تجاوز المحاولات** — بعد 3 محاولات خاطئة: "تم تجاوز عدد المحاولات. حاول بعد 5 دقائق"
  - ✅ **نجاح التحقق** — ينتقل للشاشة التالية

### 4. تأكيد التحقق (شاشة 3)
- دائرة خضراء كبيرة ✓
- عبارة "تم التحقق بنجاح"
- "أنت عميل موثوق لدى {اسم المتجر}"
- زر كبير: "عرض المنتج الرقمي 🎁"
- الجلسة صالحة لمدة 10 دقائق + مربوطة بنفس الـ IP

### 5. استلام المنتج (شاشة 4)
- شريط علوي بلون المتجر: "🎉 تم تسليم المنتج بنجاح"
- اسم المنتج ونوعه
- بطاقة البيانات الرقمية:
  - البيانات (الكود/المفتاح/الرابط) في مربع بخلفية رمادية
  - زر **نسخ** بجانب البيانات
- تعليمات الاستخدام (إذا أضافها التاجر)
- **ملاحظة:** لا يمكن إعادة فتح الرابط وعرض الكود مرة ثانية

### 6. محاولة إعادة الفتح
- إذا حاول العميل فتح الرابط مرة ثانية:
  - إذا تم التسليم: "✅ تم استلام المنتج مسبقًا"
  - إذا لم يكمل: يرجع لنموذج التحقق

### 7. ماذا لو الرابط خطأ؟
- رابط غير صالح: "الرابط غير صالح"
- رابط منتهي: "انتهت صلاحية الرابط"
- طلب ملغي: "الطلب ملغي"
- متجر غير موجود: "المتجر غير موجود"

## الأمان (Security)

| الطبقة | الآلية |
|--------|--------|
| رابط التسليم | 64 حرف عشوائي (SHA-256 hashed) - لا يُخزن raw token |
| تحقق ثنائي | رقم الطلب + رقم الجوال |
| حماية التخمين | 5 محاولات/دقيقة - 3 محاولات خاطئة = قفل 5 دقائق |
| تخصيص المخزون | Pessimistic Locking - كود واحد فقط لكل طلب |
| تشفير البيانات | AES-256 - الأكواد مشفرة في قاعدة البيانات |
| Session Token | مؤقت 10 دقائق + مربوط بـ IP |
| منع إعادة العرض | بعد product_viewed لا يمكن فتح المنتج مرة ثانية |
| عزل التجار | كل تاجر يرى منتجاته وطلباته فقط |
| إخفاء البيانات | أرقام الجوال masked في سجلات المحاولات |
| Admin middleware | صلاحيات admin منفصلة تمامًا |

## الهيكل

```
سلمها - المنصة كاملة تتكون من:
├── لوحة تحكم التاجر (Dashboard)
│   ├── الرئيسية (إحصائيات + آخر الطلبات + تنبيهات المخزون)
│   ├── المنتجات (إضافة + قائمة + إحصائيات المخزون)
│   ├── الطلبات (إنشاء + جدول + متابعة حالة التسليم)
│   ├── سجل التوصيل (لكل طلب)
│   └── الإعدادات (اسم المتجر + اللون + طريقة التحقق)
│
├── صفحة تسليم العميل (Public)
│   ├── شاشة التحميل
│   ├── نموذج التحقق (رقم الطلب + الجوال)
│   ├── تأكيد التحقق
│   ├── عرض المنتج الرقمي (كود + نسخ + تعليمات)
│   └── شاشات الخطأ (رابط خاطئ، منتهي، مسلم مسبقًا)
│
└── لوحة الأدمن
    ├── قائمة المتاجر
    ├── تفعيل/تعطيل المتاجر
    └── إحصائيات المنصة
```

## تدفق العمل الكامل (End-to-End Flow)

```
التاجر يسجل ← ينشئ منتج ← يرفع مخزون
         ↓
ينشئ طلب برقم الطلب وبيانات العميل
         ↓
يتولّد رابط تسليم آمن (يظهر مرة واحدة)
         ↓
التاجر يرسل الرابط للعميل
         ↓
العميل يفتح الرابط ← يشوف شعار المتجر
         ↓
يدخل رقم الطلب + رقم الجوال
         ↓
❌ خطأ ← إعادة محاولة (أقصى حد 3 خلال 5 دقائق)
✅ نجاح ← جلسة مؤقتة 10 دقائق
         ↓
يضغط "عرض المنتج الرقمي"
         ↓
يتم تخصيص كود من المخزون (Pessimistic Lock)
         ↓
يُفك تشفير الكود ويعرض للعميل
         ↓
يسجل في سجل التوصيل: product_viewed
         ↓
التاجر يشوف في لوحة التحكم: تم الاستلام ✅
```
